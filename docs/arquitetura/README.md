# Arquitetura

Esta pasta reúne a documentação da arquitetura do Projeto Fiscal. Seu conteúdo deve permitir que avaliadores, colaboradores e futuros mantenedores compreendam como o sistema está organizado e por que essa organização foi adotada.

## Conteúdo recomendado

- visão geral e contexto do sistema;
- módulos, camadas e responsabilidades;
- integrações e dependências externas;
- fluxos relevantes de dados e de negócio;
- requisitos e atributos de qualidade;
- estratégias de segurança, implantação e observabilidade;
- diagramas acompanhados de explicação textual e legenda.

Decisões que expliquem **por que** uma solução arquitetural foi escolhida devem ser registradas em [`../adr/`](../adr/README.md). Este diretório deve descrever principalmente o estado atual da arquitetura.

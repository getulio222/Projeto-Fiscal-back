# Diagramas

Esta pasta centraliza diagramas que apoiam a compreensão do Projeto Fiscal, incluindo contexto, contêineres, componentes, dados, implantação, sequências e fluxos de negócio.

## Diretrizes

- Cada diagrama deve declarar seu objetivo, escopo e nível de abstração.
- Elementos, relacionamentos, símbolos e cores devem possuir significado claro.
- Diagramas devem refletir o estado atual do sistema ou indicar explicitamente quando representarem uma proposta.
- Informações relevantes devem possuir uma descrição textual para manter a documentação acessível.
- Sempre que possível, os diagramas devem ser definidos como código em arquivos Markdown, por exemplo com Mermaid.
- Decisões justificadas por um diagrama devem apontar para o ADR correspondente.

Utilize nomes descritivos em `kebab-case`, como `fluxo-de-autenticacao.md`.
